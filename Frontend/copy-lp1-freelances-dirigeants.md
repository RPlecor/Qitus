## Principes de rédaction appliqués (v3)

Cette copy est écrite pour des **scanners, pas des lecteurs**. Chaque section doit fonctionner seule — un visiteur qui scrolle vite ne lit pas dans l'ordre.

Règles appliquées :
- Chaque section = une idée, un bénéfice, un argument.
- Phrases courtes. Paragraphes courts. Pas de mur de texte.
- Les mots importants sont en gras — le visiteur qui ne lit que le gras doit comprendre la proposition.
- Vendre le résultat, pas la feature. "Votre compta est faite" > "Génération d'écritures en partie double".
- Le pricing est abordé — pas caché.
- La page repousse activement les mauvais prospects (pas pour les grandes entreprises, pas un remplacement d'EC).

# BRIEF DESIGNER

## Principes non négociables (issus de la revue design)

Ces principes doivent guider chaque décision visuelle de la LP.

### 1. Blink test — 0,5 seconde

Le visiteur juge le site en une demi-seconde. Si ça a l'air cheap, il lira la meilleure copy du monde avec scepticisme. La LP doit respirer la crédibilité dès le premier écran : typographie propre, espacement généreux, palette sobre.

### 2. Zéro stock photo

Pas de groupe de professionnels souriants autour d'un laptop. Pas de poignée de main corporate. Pas d'image IA "soulless." Les visuels doivent être : des screenshots réels du produit (ou des mockups fidèles), des icônes/illustrations sur mesure pour les blocs (triple couche, 4 étapes), ou rien — un espace blanc bien géré vaut mieux qu'une image générique.

### 3. Chargement < 3 secondes sur mobile

Tester sur mobile avec 2 barres de signal, pas sur un MacBook en fibre. Chaque image compressée. Pas de plugin lourd. Pas de font exotique non-system. Lazy load sur tout ce qui est sous le fold.

### 4. Animations minimales et fonctionnelles

Hover sur les boutons. Fade-in au scroll pour le contenu. C'est tout. Pas de parallax. Pas de 3D. Pas de Spline/Rive. Si le visiteur remarque l'animation avant l'offre, c'est raté.

### 5. Écrire pour les scanners

La copy est déjà structurée pour ça, mais le design doit l'amplifier : les phrases en gras doivent visuellement ressortir (taille, poids, contraste). Les blocs de 3 (triple couche, FAQ) doivent être scannables d'un coup d'œil. Les paragraphes de plus de 3 lignes sur mobile = red flag.

### 6. Fewer pages, one clear path

C'est une single-page LP. Pas de navigation complexe. Le seul lien de nav = ancre vers les sections. Le seul CTA = liste d'attente (répété 2 fois : hero + fin). Chaque scroll amène vers le CTA, pas vers une autre page.

### 7. Différenciation visuelle

Si la LP ressemble à toutes les LP SaaS (gradient bleu, illustrations Notion-style, headline en Inter), on sera perçu comme un commodity. La direction artistique doit refléter la personnalité du produit : fiable, concret, français, pas corporate-générique. Explorer une palette qui ne soit pas le bleu SaaS standard.

### Emplacements visuels dans la copy

| Section | Visuel attendu | Contrainte |
|---|---|---|
| Hero | Screenshot/mockup du dashboard | Au-dessus du fold. Doit se charger immédiatement. |

## Hypothèses

1. **"La majorité"** = ~85 % cible technique. Pas encore mesuré. Rester flou jusqu'à données réelles.
2. **CTA "liste d'attente"** → remplacer par "Créer mon compte" si la beta ouvre avant publication.
3. **Lien EC** (Phase 8.5 livrable 3) doit être implémenté pour tenir la promesse étape 4. Sinon reformuler en "à venir".
4. **TVA** retirée du body. Réintégrer si Phase 8.5 livrable 1 est livrée avant publication.
5. **Chat comptable** = Phase 9. Ne pas promettre.
6. **"Accès prioritaire"** = engagement à honorer (early access, pricing early adopter, ou canal feedback).
7. **Bloc "pas fait pour"** mentionne les cabinets EC → cohérent avec la LP2 dédiée à venir.
8. **Pricing "à partir de"** : si le montant n'est pas décidé, mettre au minimum un ancrage qualitatif ("significativement moins cher que Pennylane").
