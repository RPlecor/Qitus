Guide utilisateur

Qitus pour les debutants

Ce guide explique Qitus comme si vous n'aviez jamais utilise de logiciel SaaS ni de logiciel comptable. Objectif : vous permettre d'utiliser Qitus de maniere autonome, page par page, sans connaitre le vocabulaire comptable au depart.

En resume

Qitus sert a importer vos transactions bancaires, les transformer en ecritures comptables, corriger les operations a revoir, rattacher les justificatifs, suivre la TVA, les rapprochements, la cloture et les documents, et preparer un dossier lisible pour votre expert-comptable.

Qitus ne remplace pas votre expert-comptable pour les arbitrages complexes. Il vous aide a preparer un dossier propre, trace et verifiable.

1. Les mots a connaitre avant de commencer

SaaS
Une application accessible dans votre navigateur. Vous cliquez dans les menus, remplissez des formulaires et validez des actions.
Tableau de bord
Page d'accueil de Qitus. Resume ce qui va bien, ce qui manque et ce qu'il faut faire ensuite.
Exercice
Periode comptable, souvent une annee (ex : du 1er janvier au 31 decembre 2025). Toutes les donnees affichees dependent de l'exercice actif.
Transaction
Ligne bancaire importee : paiement, virement, encaissement, prelevement. Exemples : paiement OVH, encaissement client, prelevement URSSAF, frais bancaire.
Ecriture comptable
Traduction comptable d'une transaction. Contient des lignes avec des comptes, des debits et des credits. Qitus cherche toujours des ecritures equilibrees.
Debit / Credit
Chaque ecriture doit avoir autant de debit que de credit. Une ecriture equilibree est normale ; une ecriture desequilibree doit etre corrigee.
Compte comptable
Code qui classe une operation. Exemples : 5121 banque, 471 attente, 44566 TVA deductible, 44571 TVA collectee.
Categorie
Classement d'une transaction dans les bons comptes. Si Qitus n'est pas certain, la transaction passe en "A verifier".
Regle de correction
Applique automatiquement une correction a des transactions similaires. Ex : toutes les transactions "OVH" → compte hebergement.
Piece justificative
Preuve d'une operation : facture, recu, contrat, releve, decision utilisateur, validation expert-comptable.
OD (Operation Diverse)
Ecriture de cloture ou d'ajustement : amortissement, CCA, provision, regularisation TVA, ecart de rapprochement. Jamais creee automatiquement sans validation.
FEC
Fichier des Ecritures Comptables. Fichier officiel que votre entreprise doit pouvoir produire en cas de controle.
TVA
Taxe sur la valeur ajoutee. Qitus distingue TVA deductible (achats), TVA collectee (ventes) et TVA nette (difference).
CA3 / CA12
Declarations TVA. CA3 : mensuelle (regime reel normal). CA12 : annuelle (regime reel simplifie). Brouillons locaux, pas de teletransmission.
Liasse fiscale
Ensemble de documents fiscaux de fin d'exercice. Source structuree locale utile pour preparer le dossier.
Expert-comptable (EC)
Personne ou cabinet qui relit, controle et valide votre dossier. Qitus prepare un dossier EC partageable en lecture seule.

2. Comment lire les statuts dans Qitus

Qitus utilise des badges de statut dans toute l'application. Voici les plus courants :

StatutSignification
A verifierVous devez regarder l'element
CorrigeUne correction a ete appliquee
ConfirmeL'element est valide pour le moment
BrouillonL'element existe mais n'est pas encore valide
ValideL'element a ete approuve
RejeteL'element a ete refuse avec une note
A regenererUn document est devenu ancien apres une modification
A jourLe document ou le controle correspond a l'etat actuel
BloquantVous devez traiter ce point avant une etape importante
WarningCe n'est pas toujours bloquant, mais il faut comprendre le point
Regle simple
Quand vous voyez un bouton d'action, Qitus attend souvent une decision : corriger, valider, rejeter, rattacher une piece, regenerer un document ou relancer un rapprochement.

3. Parcours recommande pour un debutant

Si vous ne savez pas par ou commencer, suivez cet ordre :

  1. Ouvrir Profil et verifier les informations de l'entreprise.
  2. Ouvrir Exercices et verifier l'exercice actif.
  3. Aller dans Imports et importer les transactions bancaires.
  4. Aller dans Transactions et corriger les lignes "A verifier".
  5. Aller dans Pieces pour ajouter les justificatifs importants.
  6. Aller dans Ecritures pour verifier que le journal est equilibre.
  7. Aller dans TVA si votre entreprise est au regime reel.
  8. Aller dans Rapprochements pour comparer banque, Stripe, tiers et comptes d'attente.
  9. Aller dans Controle pour traiter les points de pre-cloture.
  10. Aller dans Documents pour generer FEC, etats financiers, liasse et paquet de preuve.
  11. Aller dans Couverture EC pour voir ce qui manque encore.
  12. Aller dans Cloture quand tout est pret.
  13. Aller dans Dossier EC pour preparer le partage avec l'expert-comptable.

5. Page Tableau de bord

/dashboard

Le tableau de bord est votre page d'accueil. Il repond a trois questions : ou en est mon dossier ? que dois-je traiter maintenant ? est-ce que mes donnees sont coherentes ?

Capture d'ecran
Tableau de bord complet avec en-tete, cartes KPI, alertes et transactions recentes

En-tete

Vous voyez le nom de l'entreprise et l'exercice actif. Exemple : Bonjour - ACME DIGITAL · Exercice 2025. Si l'exercice n'est pas celui que vous voulez, allez dans Exercices.

Signal de coherence

Qitus peut afficher Exploitation coherente quand les compteurs principaux sont alignes, ou Donnees a revoir quand certains compteurs ne sont pas coherents. Ce signal ne remplace pas un controle comptable complet.

Alertes

Les alertes indiquent les actions importantes : transactions a corriger, documents a regenerer, OD a valider, rapprochements incomplets, blocages de cloture. Cliquez sur les liens d'action proposes.

9. Page Connecteurs

/connecteurs

Regroupe les connexions externes et les diagnostics d'infrastructure. Les KPI affichent l'etat Open Banking, les connexions actives, la readiness beta et le stockage.

Les connexions bancaires peuvent etre connectees, synchronisees, deconnectees ou reconnectees si le consentement est expire. Un tableau de readiness beta liste les controles d'environnement avec statut et action recommandee.

10. Page Transactions

/transactions

C'est l'une des pages les plus importantes. Elle liste les transactions bancaires importees.

Capture d'ecran
Liste des transactions avec filtres actifs, badges de statut et cartes KPI (Total, A verifier, Corrigees, Avec regle)

Filtres disponibles

Recherche (libelle, contrepartie, reference), statut (Toutes, A verifier, Categorisees, Confirmees, Corrigees, Avec regle), periode (Du/Au), compte comptable, sens (Tous, Decaissements, Encaissements) et nombre par page (25, 50, 100).

Tableau des transactions

Colonnes : Date, Libelle, Compte, Montant, Statut, Regle, Action. Des badges de filtres actifs vous aident a comprendre pourquoi certaines transactions n'apparaissent pas.

11. Page Detail Transaction

/transactions/:id

Cette page sert a comprendre et corriger une transaction precise. Navigation possible : retour a la liste filtree, transaction precedente/suivante.

Capture d'ecran
Detail d'une transaction "A verifier" : cartes (date, montant, compte, statut), suggestions, formulaire de correction

Suggestions

Qitus propose des categorisations avec badge (appliquee, suggeree, a verifier), source, comptes debit/credit, taux TVA, nature TVA, libelle et raison.

Formulaire de correction

Champs : compte debit, compte credit, libelle ecriture, taux TVA (Aucune / 0% / 2.1% / 5.5% / 10% / 20%), nature TVA, et case "Apprendre cette correction" pour creer une regle automatique.

Attention
Apres validation, Qitus peut regenerer l'ecriture, creer une regle, mettre a jour les compteurs et rendre certains documents obsoletes.

12. Page Regles

/corrections

Gere les regles de correction pour automatiser les categorisations futures. Creez une regle en indiquant une contrepartie (OVH, GOOGLE, URSSAF), un compte prefere, une condition de declenchement et une note explicative.

Le tableau affiche statut, contrepartie, compte, condition, impact (nombre de transactions concernees), note et action (desactiver/reactiver). Si une regle est trop large, desactivez-la et creez une regle plus precise.

13. Page Pieces

/pieces

Gestion des justificatifs : factures, recus, contrats, releves, decisions utilisateur. Upload possible en PDF, PNG, JPG/JPEG, TXT (max 10 Mo).

Capture d'ecran
Page Pieces avec zone d'upload, KPI (requises manquantes, non rattachees, OCR a revoir) et tableau des pieces

KPI cles

Pieces totales, requises manquantes, non rattachees (importees mais pas reliees), OCR a revoir (extraction a completer).

Les sections "Ecritures sans piece" et "Pieces sans ecriture" aident a identifier les decalages.

14. Page Revue des pieces

/pieces/revue

Guide pour fournir les justificatifs manquants. Trois onglets : A fournir (priorite haute), Recommandees (utiles mais pas toujours bloquantes), Satisfaites (exigences couvertes).

Methode recommandee
Quand vous uploadez depuis une exigence precise, Qitus rattache automatiquement la piece au bon element. C'est la methode la plus fiable pour les debutants.

15. Page Detail Piece

/pieces/:id

Verifier, corriger et rattacher une piece. Le formulaire de metadonnees comprend : fournisseur, date facture, numero, HT, TVA, TTC, devise, texte extrait (corrigeable manuellement).

Qitus propose des suggestions de rattachement avec un score base sur montant, date, fournisseur et type de preuve attendu. Vous pouvez aussi rattacher la piece a une exigence active ou l'archiver.

16. Page Ecritures

/ecritures

Le journal comptable — traduction comptable des transactions et OD.

Capture d'ecran
Journal comptable avec KPI (Debit, Credit, Equilibre) et tableau des ecritures

Alertes

Journal audit (exportable et equilibre ?), evidence (ecritures sans pieces ?), anomalies (ecriture sans ligne, debit+credit simultane, ligne a zero, desequilibre).

Filtres

Journal (BQ, OD), source (import, cloture, manuel), compte, recherche, periode, nombre par page.

Tableau

Colonnes : N°, Date, Journal, Source, Libelle, Compte, Debit, Credit. Export CSV disponible (ne remplace pas le FEC officiel).

17. Page Documents

/documents

Generation et telechargement des documents comptables : FEC, etats financiers (balance, bilan, compte de resultat), liasse fiscale, paquet de preuve.

Capture d'ecran
Cartes de generation des documents : FEC, Etats financiers, Liasse fiscale, Paquet de preuve
Important
Avant de generer le FEC, corrigez les transactions bloquantes si possible. Le paquet de preuve rassemble FEC, etats, liasse, pieces, rapports de rapprochement, workpapers, OD, audit et manifest.

18. Page Controle

/controle

Points de controle de pre-cloture. Transforme les alertes en actions suivables. KPI : statut global, blocages, ouverts, OD proposees, documents, pieces.

Les sections couvrent : blocages (qui empechent une etape importante), justificatifs manquants, OD proposees (brouillon, validee, rejetee), documents (a jour ou a regenerer), points a revoir (warnings) et points deja traites (historique).

19. Page Detail OD

/controle/od/:proposalKey

Relire une operation diverse avant validation. Une OD brouillon ne cree pas encore d'ecriture.

Capture d'ecran
Detail OD : KPI (type, montant, fraicheur), hypotheses, lignes debit/credit, boutons Valider / Rejeter

Types d'OD

CCA (charges constatees d'avance), amortissement, IS (impot sur les societes), FNP, provision, variation de stock.

Actions

Enregistrer les hypotheses, recalculer, valider l'OD (cree l'ecriture comptable), rejeter (note requise), reouvrir une OD rejetee.

Action importante
Valider une OD modifie la comptabilite. Verifiez hypotheses, calcul, lignes debit/credit, piece liee et impact resultat/bilan avant de valider.

20. Page TVA

/tva

Suivi de la TVA. KPI : regime (franchise, reel simplifie, reel normal), TVA collectee (44571), TVA deductible (44566), net.

Les controles TVA listent les problemes : taux manquant, nature manquante, declaration obsolete, compte TVA incoherent. Des tableaux par taux et par nature detaillent la base HT, deductible, collectee et net. La section declarations brouillon affiche CA3 ou CA12 avec periode, statut, fraicheur et TVA nette.

21. Page Revue TVA

/tva/revue

Traitement un par un des problemes TVA. KPI : points, bloquants, alertes. La resolution passe par le flux de correction transaction pour eviter les ecritures TVA incoherentes.

22. Page Detail Declaration TVA

/tva/:declarationId

Declaration TVA brouillon. Actions : retour, telecharger, regenerer si obsolete. Affiche les cases brouillon (base HT, TVA collectee, deductible, autoliquidation, nette), les sources par taux, les controles lies et une comparaison avec le journal pour detecter les ecarts.

23. Page Rapprochements

/rapprochements

Compare les donnees comptables avec d'autres sources : banque, Stripe, comptes clients/fournisseurs, comptes d'attente. KPI : blocages, avertissements, progression banque, fraicheur.

24. Page Rapprochement Banque

/rapprochements/banque

Rapproche transactions bancaires et lignes comptables du compte 5121. KPI : statut ligne a ligne, matches, issues ouvertes, solde releve.

Saisissez le solde et la date du releve bancaire. Le tableau des matches affiche : gauche (transaction bancaire), droite (ligne comptable), statut (auto-matche, confirme, unmatched, ignore, difference), ecart, note et action.

Confirmation
Ne cochez "Confirmer si aucun ecart" que si vous avez verifie que le solde comptable et le releve concordent.

25. Page Rapprochement Stripe

/rapprochements/stripe

Rapproche payouts Stripe et transactions bancaires. Actions : importer fixture, synchroniser connecteur, lancer rapprochement. Le tableau des matches affiche payout, banque, statut et ecart. Les evenements Stripe detaillent date, type (charge, fee, refund, dispute, payout), brut, frais et net.

26. Page Revue Rapprochements

/rapprochements/revue

Liste toutes les issues de rapprochement. Filtres par type (banque, Stripe, tiers, attente), statut (open, resolved, ignored) et severite (blocking, warning).

Audit trail
Une resolution ou un ignore doit avoir une note. Cette note sert a l'audit.

27. Page Couverture EC

/couverture

Indique si le dossier est suffisamment complet pour un expert-comptable. Ne certifie pas le dossier — montre les zones couvertes, partielles ou manquantes.

KPI : score global, domaines couverts, partiels, manquants. Les risques principaux listent justificatifs manquants, TVA incomplete, rapprochements non traites, dossier EC non valide. Le tableau des domaines affiche statut, risque, resume et phase roadmap.

30. Page OD de Cloture

/cloture/od

31. Page Workpapers

/cloture/workpapers/:kind

Documents de travail par domaine de cloture : FNP, FAE, PCA/CCA, stocks, provisions, emprunts, paie, TVA, IS, ecarts de rapprochement.

Chaque workpaper a un titre, un statut (brouillon → pret → archive), des hypotheses, un calcul, une piece attendue et une note. "Pret" signifie pret a generer une proposition d'OD, pas valide comptablement.

32. Page Immobilisations

/immobilisations

Liste les biens durables a amortir : ordinateur, mobilier, materiel, equipement. Formulaire : libelle, compte d'immobilisation, date acquisition, montant, duree en annees, compte amortissement, compte dotation.

Une immobilisation peut alimenter les propositions d'amortissement en cloture.

33. Page Dossier EC

/dossier-ec

Prepare le dossier pour l'expert-comptable en regroupant les preuves. KPI : score dossier, sections pretes, blocages, revue EC.

Capture d'ecran
Dossier EC avec score, sections (FEC, TVA, pieces, rapprochements), readiness et actions de partage

Actions : preparer le dossier (snapshot), partager au cabinet (lien lecture seule), exporter dossier final. Le tableau des sections affiche etat (pret, partiel, bloque, obsolete), risque et manques.

34. Page Revue Dossier EC

/dossier-ec/revue

Gere les demandes et commentaires de l'expert-comptable. Champs de creation : section, severite (info, warning, blocking), titre, detail.

Regles importantes
Une demande bloquante ouverte empeche la validation finale. Une resolution ou un waiver doit avoir une note. L'expert peut commenter et valider, mais ne modifie pas la comptabilite.

35. Page Snapshots Dossier EC

/dossier-ec/snapshots

Un snapshot fige l'etat du dossier a un instant donne. Il devient obsolete si la comptabilite est modifiee apres. Creez un nouveau snapshot apres : correction de transaction, validation d'OD, ajout/suppression de piece, regeneration de document, nouveau rapprochement, modification TVA ou reouverture de cloture.

36. Portail partage Expert-Comptable

/shared/:token

Page en lecture seule pour l'expert-comptable. Il peut consulter le dossier, voir les documents, lire les pieces, examiner TVA/rapprochements/cloture, creer des demandes, commenter et valider. Il ne peut pas modifier une transaction, valider une OD, creer une ecriture, regenerer un document ou changer la cloture.

37. Page Chat

/chat

Assistant comptable contextuel en lecture seule. Il peut expliquer les donnees et orienter vers les pages utiles. Il ne cree pas d'ecriture, de document, d'OD ou de correction.

Exemples de questions : "Pourquoi la cloture est bloquee ?", "Quelles transactions restent a corriger ?", "Quels documents sont a regenerer ?"

38. Page Abonnement

/abonnement

Plan (SOLO, ENTREPRISE, ENTREPRISE_PLUS), statut, quotas IA et imports, etat des droits, changement de plan et portail client Stripe.

39. Page Exercices

/exercices

L'exercice actif controle presque toutes les donnees affichees. Si vous ne voyez pas les bonnes transactions, verifiez l'exercice actif.

Attention
Changer l'exercice actif modifie ce que vous voyez dans toutes les pages. Qitus refuse les dates qui se chevauchent avec un autre exercice.

40. Page Profil

/profil

Informations de l'entreprise (nom, forme juridique, SIREN, regime TVA, exigibilite TVA, impot IS/IR) et actions RGPD (exporter, anonymiser, demander suppression).

Regime TVA
Changer certains champs fiscaux peut rendre des documents obsoletes. Si vous ne savez pas quel regime ou quelle exigibilite choisir, demandez a votre expert-comptable avant de modifier.

41. Page Demo

/demo (mode developpement uniquement)

Charge un scenario de demonstration. Datasets : qonto_mvp (parcours de base), multi_bank (multi-banques CSV), regime_reel_tva (TVA regime reel), closing_beta (cloture complete).

Destructif
Charger un dataset demo reinitialise les donnees locales. Vous devez cocher la case de confirmation avant de pouvoir charger.

42. Que faire quand Qitus affiche "A regenerer"

Un document devient A regenerer quand les donnees qui ont servi a le produire ont change (transaction corrigee, OD validee, piece rattachee, TVA changee, rapprochement relance, workpaper modifie, profil fiscal modifie).

  1. Comprendre ce qui a change.
  2. Aller dans Documents.
  3. Regenerer le document concerne.
  4. Verifier que l'etat repasse a A jour.

43. Que faire quand une transaction est "A verifier"

  1. Aller dans Transactions.
  2. Filtrer sur A verifier.
  3. Ouvrir la premiere transaction.
  4. Lire les suggestions.

46. Regles de prudence

Ne validez pas une OD sans comprendre son effet
Une OD validee cree une vraie ecriture comptable. Relisez hypotheses, calcul, lignes debit/credit, piece liee, impact resultat/bilan.
Ne masquez pas un blocage sans note
Si vous ignorez un point, ajoutez une note claire. Exemple : "Ignore car operation hors exercice, verifie avec EC le 21/05/2026".
Ne changez pas le regime TVA au hasard
Le regime TVA influence les ecritures, declarations et documents. Demandez conseil si vous n'etes pas certain.
Ne supprimez pas les fichiers locaux manuellement
Utilisez les actions d'archivage ou RGPD prevues. Un fichier supprime manuellement peut signaler un artefact manquant.
Travaillez exercice par exercice
Verifiez toujours l'exercice actif avant de corriger ou generer des documents.

48. Parcours ultra simple pour votre premiere session

Si vous voulez juste faire un premier tour :

  1. Ouvrez Tableau de bord.
  2. Notez le nombre de transactions a verifier.
  3. Ouvrez Transactions.
  4. Filtrez A verifier.
  5. Corrigez une transaction.
  6. Ouvrez Ecritures.
  7. Verifiez que l'ecriture existe.
  8. Ouvrez Pieces.
  9. Ajoutez une piece simple.
  10. Rattachez-la a la transaction.
  11. Ouvrez Documents.
  12. Generez le FEC.
  13. Ouvrez Couverture EC.
  14. Regardez ce qui manque encore.
Resultat
Avec ce parcours, vous aurez compris la logique centrale de Qitus : importer → corriger → prouver → generer → verifier.

49. En cas de doute

  1. Ne validez pas une ecriture ou une OD au hasard.
  2. Lisez la raison affichee par Qitus.
  3. Regardez l'historique dans Activite.
  4. Consultez la piece justificative.
  5. Utilisez le Chat pour une explication en lecture seule.
  6. Demandez a votre expert-comptable si le point a un impact fiscal ou juridique.
Philosophie Qitus
Qitus est concu pour rendre les decisions visibles et auditables. Le bon reflexe est de documenter vos choix avec des notes et des pieces.
