Message beta : Qitus est une beta comptable pour centraliser imports, pièces, TVA, clôture, rapprochements et dossier expert-comptable. Les connecteurs et la facture électronique peuvent être testés dans un environnement interne contrôlé. Les écritures restent validées par l'utilisateur.

Bloc D
Parcours MVP comptable
Import CSV → catégorisation → corrections → règles → écritures → export journal → activité → notifications.

Blocs M→O — Ops & compliance

Bloc M
Règles & impacts de changement
Alertes TVA, impacts documents/FEC/rapprochements/dossier EC après modification. Pas de doublon d'alertes.
7 étapes
L'utilisateur sait quoi relancer après un changement, sans modification silencieuse du passé.
Bloc N
Chat, abonnement & limites
Chat fake en staging, billing en stub, limites lisibles, pas de confusion Stripe billing / Stripe rapprochement.
5 étapes

Bloc O
Sécurité & conformité beta
Secrets protégés, cookies sécurisés, webhooks signés, rate limit, mutations bloquées si clôturé, RGPD, anonymisation.
10 étapes
Qitus testable par des utilisateurs beta sans exposition de secrets ni comportement destructif.

Go / No-Go beta

Go — Ouvrir la beta si :

  • Render déploie automatiquement depuis GitHub
  • L'inscription Clerk fonctionne
  • L'onboarding fonctionne
  • La base est persistante
  • Les backups PostgreSQL sont actifs
  • Le parcours MVP passe
  • Le reset import est possible sans SQL
  • Documents et pièces persistants ou limités à des tests sans données sensibles
  • Les connecteurs test ne sont pas visibles hors mode interne
  • Les secrets ne sortent pas dans les statuts
  • Les erreurs utilisateur sont lisibles

No-Go — Ne pas ouvrir si :

  • /dashboard inaccessible après login
  • Les imports nécessitent du SQL pour être corrigés
  • Fichiers sur disque éphémère avec de vraies données
  • Backups inconnus
  • Clerk affiche une expérience non brandée Qitus
  • Les connecteurs test sont visibles comme connecteurs produit
  • Génération d'écriture automatique sans validation utilisateur
  • Les secrets apparaissent dans une page, API ou log
