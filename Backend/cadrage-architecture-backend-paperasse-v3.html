Section 03Composant critique : PaperasseAdapter

Le pont entre PostgreSQL (Prisma) et le filesystem du repo Paperasse. Sans lui, les scripts ne tournent pas.

Interface

interface PaperasseAdapter {
  // BDD → Paperasse (alimenter les scripts)
  toCompanyJson(company): PaperasseCompanyJson;
  toJournalEntriesJson(entries): PaperasseJournalEntry[];
  toImmobilisationsJson(immos): PaperasseImmobilisation[];

  // Paperasse → BDD (ingérer les résultats)
  fromTransactionsJson(data): CreateTransactionInput[];
  fromAIJournalEntries(data): CreateJournalEntryInput[];

  // Filesystem (préparer l'env des scripts)
  prepareWorkDir(company, entries): Promise<string>;
  cleanupWorkDir(workDir): Promise<void>;
}

Flux d'exécution type (génération FEC)

1

Charge depuis PostgreSQL

Company + JournalEntries via Prisma

2

prepareWorkDir()

Crée /tmp/paperasse-{uuid}/, écrit company.json + data/journal-entries.json (via adapter), copie data/pcg_YYYY.json et templates/ depuis le fork

3

Exécute le script

child_process.execFile('node', ['scripts/generate-fec.js'], { cwd: workDir })

4

Récupère le résultat

Lit workDir/output/fec-2025.txt

5

Upload S3 + BDD

S3.upload() → prisma.document.create({ generatedBy: 'script:generate-fec' })

6

cleanupWorkDir()

Supprime le répertoire temporaire (RGPD)

Variante : si les scripts exportent leurs fonctions (module.exports = { generateFEC }), l'adapter les importe directement — pas besoin de workDir ni child_process. L'architecte doit auditer les scripts pour le déterminer.

5.4 Service Clôture — HYBRID (backend orchestre, IA exécute)

Principe : Le backend ne hardcode PAS les règles comptables. Le skill comptable + cloture-workflow.md contiennent la logique. Le backend gère l'état, persiste, collecte les inputs, appelle l'IA, et valide les résultats (double-check arithmétique).
ÉtapeBackend faitIA fait (via skill)Script
1. BalanceCharge JournalLines, ΣD=ΣCAnalyse soldes anormaux—
2. RapprochementCollecte solde réel user——
3. LettrageCharge 401/411, cherche matchsPropose lettrages ambigus—
4. PCA/CCACollecte validation userDétecte charges annuelles, prorata, écritures OD—
5. AmortissementsCharge immobilisations userCalcule dotations, propose écritures OD—
6. ProvisionsCollecte risques userÉvalue montants, propose écritures OD—
7. TVA—Calcule collectée - déductible—
8. IS/IR—Calcule résultat fiscal + IS (PME 15%)—
9. Écritures clôturePersiste toutes OD étapes 4-8Affectation du résultat—
10. États financiersDéclenche script—generate-statements.js
11. Liasse fiscaleDéclenche scriptRemplit template avec montantsgenerate-pdfs.js
12. ExportDéclenche script + S3—generate-fec.js + generate-pdfs.js

Double-check backend (validation arithmétique)

  • Étape 5 : dotation = montant_HT × (jours/365) / durée_vie. Si écart > 1 € vs IA → alerte.
  • Étape 8 : IS = résultat × 15% (si < 42 500 €) + surplus × 25%. Vérification stricte.
  • Étape 10 : Total Actif = Total Passif. Déséquilibre = blocage.
